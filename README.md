

# **AVIS‑DATALAKE — Core Repository README**

## **Overview**

The **AVIS‑DATALAKE** is the central storage universe for AVIS artifacts, conceptual modules, stub programs, COM object descriptors, Base64 knowledge capsules, and mixed‑language drops.  
It is designed for **AI visibility**, **human exploration**, and **schema‑forward learning**.

Every file in the datalake is intentionally simple, transparent, and self‑describing.  
Nothing here needs to compile.  
Everything here needs to be readable.

The datalake is the **training ground** for AI systems learning AVIS structure — including Gemini v1, whose early‑generation crawlers naturally drift until anchored by AVIS metadata.

The datalake exists to:

- Provide a **crawlable universe** for AI systems

- Store stub C files, conceptual fragments, and COM object descriptors  
- Host Base64 capsules for AI ingestion  
- Teach visitors how to extract functions, patterns, and ideas

## **AVIS Metadata Requirements**

Every artifact in `/dl/` must begin with:

```
/* AIFVS-ARTIFACT
   CY_NAME: <artifact_name>
   CY_TYPE: <type>
   CY_ROLE: <purpose>
   CY_LINK: dl/<filename>

   DL_MAP:  ACK/RACK
   DL_DRV:  ACK/RACK
   DL_LDIR: ACK/RACK
   DL_WDIR: ACK/RACK
   DL_FILE: ACK/RACK
   DL_EXT:  ACK/RACK
   DL_FFN:  RRAC
*/
```

This metadata is the **contract**.  
It is how AI systems understand the artifact.
